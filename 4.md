Kelvin brought up working on medians of both lists. Valid.

Thought about binary search on both lists to try to get median pointer values closer to each other, and when magnitude reverts, we know solution is the bigger number in the previous position. For even numbers, that's mean between bigger number of last and smaller number of current.

Realized this solution takes advantage of how count of sum of numbers before and including median pointers = (m+n)/2, so we really need only 1 binary search, and the 2nd median pointer reacts by shifting the same amount of indices.
